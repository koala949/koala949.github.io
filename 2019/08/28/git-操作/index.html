git命令

2019-08-28
git

从远程拉取代码及相关操作

git clone <remote_url>  
git checkout –b <newBranch>  
git add <file>  
git commit –m “commit information”  
git pull origin <remote_branch>  
git push origin <local_branch>:<remote_branch>

撤销操作分三种情况：

第一种：没有执行add指令
git checkout -- <file>
第二种：执行了add指令，未执行commit指令
git reset HEAD <file> 暂存区内容回退
git checkout -- <file> 丢弃工作区修改
第三种：执行了commit指令，未提交到远程
git reset --hard <commit_id>版本回退

查看提交历史

git log
git log --pretty=oneline 一行显示

版本回退

git reset --hard <commit_id>
(HEAD指向当前版本，HEAD^上个版本，HEAD^^上上个版本……)
git reset --soft e79fcfb: 回退到某次提交，并且把commit的内容撤回到暂存区

重返未来

git reflog查看命令历史

对比差异

git diff 查看工作区与版本库中的不同
git diff HEAD -- <file>查看某个文件的不同

文件被提交到版本库，想删除.

第一步：删除工作区文件
rm <file>
第二步：删除版本库中的文件
git rm <file>
git commit --m “remove file”

解决冲突

执行 git merge <branch name> ，如果发生冲突，先解决冲突，再提交

查看分支合并图：
git log --graph --pretty=oneline --abbrev-commit

分支管理

创建分支

git checkout -b <branch>

查看分支

git branch 查看本地分支
git branch -r 查看远程分支
git branch -a 查看所有分支

切换分支

git checkout <branch>

删除分支

git branch -d <branch>

合并分支

git merge <branch>

暂存当前分支

git stash
git stash [save "msg"]：储藏当前暂存的文件，[并提交储藏信息]

查看stash内容

git stash list 存储列表

恢复工作现场

方式一：
git stash apply 仅恢复
git stash apply stash@{0}: 应用某次储藏(不会删除那一次)
git stash drop 删除
方式二：
git stash pop 恢复且删除，（应用并弹出栈顶的储藏）

练习

==> git branch xhy 创建分支 (相当于复制一份)
==> git branch 查看本地分支
==> git branch -a 查看所有分支
==> git checkout xhy 切换分支
==> 修改文件后
==> git add .
==> git commit -m “add”
==> git checkout master
==> git merge xhy
==> git pull
==> git push
==> git checkout xhy
==> git pull origin master ：推送到远程分支之后，就要回到你自己的分支继续开发，为了获取到远程分支的最新更改，所以要重新pull一下
