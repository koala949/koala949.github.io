dva笔记

2019-09-11
react, vue

综述

dva= react-router + redux + redux-saga + fetch

简单运用：

dva api

  1. model:
    ===> namespace每个model的身份标识。
    ===> state保存的引用状态数据。
    ===> action事件：effects异步操作、reducers同步操作。
    ===> connect绑定state到view
    ===> dispatch发送action事件到state (此时需要找到对应的model)

  2. 对比vue项目
    Vue项目 = vuex + vue-router + axios
    React项目 = redux + react-router + redux-saga +fetch = dva

    Vue的路由模式有hash,history,默认hash，dva也是，默认hashHistory，可选BrowserHistory和 createMemoryHistory，因为本质上还是单页，使用BrowserHistory还需要配置服务器Nginx或者Tomcat。
    默认dva工程下需要 npm install history 才可以切换到browserHistory模式。

    Vue一般接口调用封装接口然后再组件内调用，dva选择在model中实现，这是他们模式的最大区别，model能进行异步操作，vuex的model只能进行同步操作，同步操作在组件内完成。

dva中effects

通常I/O操作，数据库读写，以key/value格式定义effect,不直接修改state，通常通过generate yield以及saga里面的常用call，put, takeEvery,takeLatest,take,将异步转为同步写法。

generator函数：yield中断：
===>call 触发异步操作
===>put 相当于 dispatch 触发reducer 改变state
===>select 从state中获取相关的数据
===>takeEvery 监听action的每次变化执行（默认）
===>takeLatest监听action最近一次的变化
===>take 监听一次action留着，后面执行动作。
